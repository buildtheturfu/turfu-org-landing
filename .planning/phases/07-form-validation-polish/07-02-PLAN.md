**CRITICAL CONSIDERATION:** The current ArticleEditor has a complex relationship between:
1. Form fields (title, category, tags) parsed from frontmatter
2. rawContent textarea that contains the full markdown with frontmatter

Two approaches:
A) Keep textarea as source of truth, parse values from it, use form for validation display only
B) Make form fields source of truth, sync back to rawContent

Recommend approach A with modification: Add a dedicated title input (separate from textarea) that syncs to rawContent. This allows proper validation UX. Other frontmatter fields (category, tags) already have dedicated inputs.
